# 家庭负荷预测系统 · 使用手册

> 基于深度学习时序模型的桌面端家庭用电量预测工具。导入一段历史用电 CSV,系统给出未来 1 天 / 7 天的逐小时用电量曲线。

---

## 一、安装与启动

1. **解压** `HouseholdLoadForecast-release.zip` 到任意目录(建议避免路径含中文空格以外的特殊字符)。
2. 进入 `HouseholdLoadForecast\` 文件夹,**双击** `HouseholdLoadForecast.exe`。
3. 首次启动会在 exe 同目录自动创建 `data\`、`runtime\` 两个文件夹,分别存放导入的 CSV 与账号/历史记录。

**系统要求:** Windows 10 x64 及以上,无需额外安装 Python、PyTorch 或任何依赖。

**SmartScreen 拦截怎么办?** 右键 exe → 属性 → 勾选"解除锁定"(Unblock) → 确定,再双击即可。

---

## 二、注册 / 登录

启动后会弹出登录窗口:

- **首次使用:** 点击「注册」→ 填写用户名(3–32 字符)与密码(至少 8 位)→ 确定 → 返回登录。
- **已注册:** 输入用户名密码 → 登录,进入主界面。

账号信息保存在本机 `runtime\users.json`,密码仅保存 PBKDF2-SHA256 摘要,不会上传到任何服务器。

---

## 三、主界面:四个工作台

登录后主窗口顶部有 4 个页签,从左到右依次是:

| 页签 | 作用 |
|------|------|
| 数据工作台 | 导入历史用电 CSV,预处理为逐小时序列 |
| 预测工作台 | 选择模型与预测周期,运行预测并导出结果 |
| 历史记录 | 查看本账号的历史预测任务 |
| 用户 | 修改密码 / 退出登录 / 注销账号 |

---

## 四、数据工作台

### 支持的 CSV 格式

- **必填列:** `WHE`(总用电量,kWh)
- **可选列:** `date`(时间戳)、`OT`(气温)、`Wind`(风速)、`RH`(相对湿度)
- **编码:** UTF-8
- **目标频率:** 小时级(1h)。若 CSV 为更细粒度(如 15 分钟),系统会自动聚合到小时。
- **建议至少 256 个小时的历史数据**(约 11 天),否则模型会因序列太短报错。

### 操作步骤

1. 点击 **「下载数据集模板」**(首次用建议先下载看列名格式),把模板另存到本地。
2. 把自己的数据按模板列名整理好,保存为 UTF-8 CSV。
3. 点击 **「选择 CSV 文件…」** → 选择你的文件。
4. 系统会执行:**列校验 → 时间规范化 → 小时级重采样**。成功后:
   - 上方状态条显示「处理成功:xxx.csv(小时级 N 点)」
   - 下方以表格形式展示前几行
   - 点击 **「数据可视化」** 可切换为 WHE 曲线图
   - 底部显示 WHE 的均值 / 最大值 / 最小值
5. 数据处理成功后,**会自动同步到预测工作台**,无需手动传递。

### 常见错误

| 错误提示 | 原因与处理 |
|----------|----------|
| 「无法以 UTF-8 解码该文件」 | CSV 用了 GBK / GB18030 等编码 → 用 Excel 或 VS Code 另存为 UTF-8 |
| 「CSV 列校验失败,缺少必填列」 | 缺 `WHE` 列 → 按模板补上 |
| 「缺少时间列」 | CSV 没有 `date` 列 → 推荐改用带 `date` 列的模板 |
| 「日期列无效」 | `date` 列有不能解析的字符串 → 统一成 `YYYY-MM-DD HH:MM:SS` |

## 五、预测工作台

### 三个下拉框的含义

| 下拉框 | 选项 | 说明 |
|--------|------|------|
| **户型** | 单户 / 多户 | 单户对应 AMPds 数据集训练的模型;多户对应 London B0 |
| **模型** | DC-iTransformer / PatchTST / TimeXer | 三种时序深度模型,精度/速度各有侧重 |
| **预测周期** | 1 天 / 7 天 | 1 天 = 未来 24 小时;7 天 = 未来 168 小时 |

### 操作步骤

1. 切换到「预测工作台」。前提是已在数据工作台成功导入一份 CSV。
2. 选好 **户型 + 模型 + 预测周期**,点击 **「开始预测」**。
3. 预测通常在 1–5 秒内完成(纯 CPU 推理),完成后:
   - 下方「结果摘要」给出预测段的均值 / 最大值 / 最小值
   - 上方图表展示:**蓝色** = 历史真实值(最近 256 小时),**橙色** = 未来预测值,中间有虚线分界
4. 需要留存预测结果时点击 **「导出预测表格」**:
   - 选择保存路径与文件名(默认 `forecast_<户型>_<模型>_<周期>_<时间戳>.csv`)
   - 导出为 UTF-8-BOM 编码 CSV,包含「日期」「用电量」两列
   - 若历史数据有时间索引,日期列会从历史末尾逐小时递增;否则是纯整数序号

### 常见错误

| 错误提示 | 原因与处理 |
|----------|----------|
| 「当前还未上传数据集」 | 切回数据工作台先导入 CSV |
| 「历史序列长度不足:至少需要 256 个点」 | 输入数据太短 → 提供更长的历史(至少约 11 天) |
| 「该户型下的该模型资源尚未接入」 | 此户型 × 模型的 checkpoint 暂未就位(罕见) |
| 「预测失败,XxxError: ...」(带 traceback) | 遇到意外错误 → 对话框会直接显示调用栈,把文本反馈给开发者即可 |

## 六、历史记录

每次成功预测都会自动追加一条记录,字段包括:记录 ID、创建时间、户型、模型、周期、导出路径。

- **刷新**:重新读取数据库(极少用,UI 已自动刷新)
- **删除选中**:勾选行后点此按钮,可删除单条或多条
- **一键清空**:清空当前账号的全部记录(不可恢复,请谨慎)
- **复制路径**:右键「输出路径」单元格 → 复制路径,粘贴到资源管理器即可打开文件所在文件夹

历史记录保存在本机 `runtime\forecast.db`(SQLite)。不同账号的记录相互隔离。

---

## 七、用户中心

- **修改密码**:输入原密码 + 两次新密码 → 成功后会自动登出,需用新密码重新登录
- **退出登录**:回到登录窗口,当前会话清空
- **注销账号**:**永久删除**当前账号及其所有历史预测记录,不可恢复;会弹出二次确认

---

## 八、数据目录说明

解压目录结构大致如下:

```
HouseholdLoadForecast\
├── HouseholdLoadForecast.exe        ← 主程序,双击启动
├── USER_GUIDE.md                    ← 本手册
├── README.md                        ← 开发者向说明(可略过)
├── data\                            ← 你导入的 CSV 放这里(首次启动自动创建)
├── runtime\
│   ├── users.json                   ← 本机账户库
│   └── forecast.db                  ← 历史预测记录
└── _internal\                       ← 运行时依赖(勿删、勿改)
    ├── resources\
    │   ├── checkpoints\             ← 模型权重
    │   ├── configs\                 ← 模型注册表
    │   └── templates\               ← CSV 模板
    ├── torch\ ...                   ← PyTorch 运行时
    └── PyQt5\ ...                   ← Qt 界面运行时
```

**迁移 / 备份:** 想把软件连同账号、历史一起迁到另一台电脑,直接拷贝整个 `HouseholdLoadForecast\` 目录即可。

**重置:** 想清空所有本地数据重新开始,删掉 `runtime\` 下的 `users.json` 和 `forecast.db` 即可(下次启动会自动重建)。

## 九、常见问题速查

**Q:首次启动很慢?**
A:PyInstaller onedir 包首次启动需要 5–10 秒加载 torch / PyQt5 的 DLL,后续正常。

**Q:预测结果波动很大 / 完全不像?**
A:当前使用的是通用数据集(AMPds / London B0)上预训练的权重,和你家的用电模式可能不完全匹配,仅供趋势参考;后续可接入自训练权重。

**Q:能预测更久吗(例如 30 天)?**
A:当前 checkpoint 仅提供 1 天 / 7 天两档,对应模型训练时锁定的 `pred_len`,不能随意拉长;若需更长预测跨度,需要重新训练并替换 checkpoint。

**Q:可以在 macOS / Linux 跑吗?**
A:当前发布包仅 Windows x64。其他平台需从源码重新打包(另见开发者说明)。

**Q:杀毒软件拦截?**
A:PyInstaller 打的 exe 偶有误报,常见是 Windows Defender 标记为未签名。右键 exe → 属性 → 解除锁定,或将解压目录加入排除项。
