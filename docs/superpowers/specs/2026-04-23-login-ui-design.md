# 登录界面美化与默认记住密码设计

日期：2026-04-23

## 1. 背景与目标

当前系统已经具备本地多账号注册、登录、改密、注销等基础能力，但登录页、注册弹窗和主界面整体视觉较为朴素，缺少统一主题层；同时登录流程不支持“默认记住密码”，每次启动都需要重新输入账号密码。

本次设计目标：

- 将登录页、注册弹窗、主界面统一为“数据科技风”视觉语言
- 新增“默认记住密码”能力
- 下次启动时自动回填用户名和密码，但不自动登录
- 保持现有本地账号体系、密码哈希规则和主流程不变
- 尽量以主题层和小范围结构优化完成升级，避免大改页面逻辑

## 2. 已确认决策

- 视觉方案：采用方案 B，单卡片数据科技风
- 登录体验：记住用户名和密码，重新打开应用时自动填充，但仍需手动点击“登录”
- 记住密码勾选框：默认勾选
- 美化范围：登录页、注册弹窗、主界面顶部/标签页，相关页面顺手统一视觉风格

## 3. 范围

本次包含：

- 登录页视觉重做
- 注册弹窗视觉统一
- 主窗口与标签页主题统一
- 历史记录、用户中心、数据页、预测页的常用控件样式统一
- 新增 remembered login 本地配置读写
- 删除账号时同步清理 remembered login

本次不包含：

- 改造账号认证算法
- 自动登录
- 远程账号体系
- 引入系统钥匙串、Windows Credential Manager 或第三方安全存储
- 重写现有页面的业务流程与布局结构

## 4. 用户体验设计

### 4.1 登录页

登录页采用单卡片居中布局：

- 背景使用深蓝到青色的渐变，营造数据科技感
- 中心卡片使用深色半透明面板、细边框、圆角和阴影
- 顶部展示系统名称与一句简短说明
- 表单区包含用户名、密码、记住密码、登录、注册入口

交互规则：

- 首次打开登录页时，“记住密码”默认勾选
- 若存在 remembered login 数据，则自动回填用户名和密码
- 点击“登录”时仍走现有认证逻辑
- 登录失败时不清除用户名；密码按现有交互清空并重新聚焦
- 若用户取消勾选“记住密码”，则本地 remembered login 数据立即清空
- 若用户登录成功且勾选为选中状态，则保存当前用户名与密码到 remembered login

密码显示/隐藏：

- 保留当前显示/隐藏能力
- 不再依赖 emoji 作为按钮文案，改为稳定的文本或标准图标方案
- 本次直接采用简短文本切换，按钮文案固定为“显示”/“隐藏”，避免编码和字体兼容问题

### 4.2 注册弹窗

注册弹窗与登录页复用同一套主题语言：

- 深色卡片、统一输入框、统一主次按钮
- 标题、说明、表单间距和按钮组风格与登录页一致

交互保持现有逻辑：

- 注册成功后关闭弹窗
- 返回登录页时自动回填刚注册的用户名
- 密码不保留，焦点落在密码框，便于继续登录

### 4.3 主界面

主界面以“统一主题层 + 轻量结构整理”为主，不推翻当前工作台结构。

保留现有四个页签：

- 数据工作台
- 预测工作台
- 历史记录
- 用户中心

视觉升级策略：

- `QMainWindow`、`QTabWidget`、状态栏、按钮、输入框、表格等控件统一为深蓝青色主题
- 顶部标签页强化选中态、悬停态和留白，让页签更像专业工作台导航
- 每个页面统一为“标题 + 简短说明 + 主操作区”的视觉节奏
- 状态信息、统计信息、结果摘要、用户信息采用卡片式容器承载
- 表格、图表配色与主题统一，但不改业务逻辑
- 危险操作按钮使用独立警示色，与主按钮区分

## 5. 数据与存储设计

### 5.1 现有认证边界

以下部分保持不变：

- `services/auth_service.py` 中的注册、登录、改密规则
- `runtime/users.json` 中的用户记录结构
- PBKDF2-SHA256 密码哈希方案

也就是说，“记住密码”只属于 UI 便利功能，不参与认证规则判断。

### 5.2 新增 remembered login 存储

新增文件：

- `runtime/remembered_login.json`

采用以下固定结构：

```json
{
  "remember_password": true,
  "username": "example_user",
  "password": "plain-text-for-local-prefill",
  "updated_at": "2026-04-23T19:00:00"
}
```

说明：

- 该文件仅用于本机桌面端回填登录表单
- 为满足“下次启动自动回填密码”的需求，密码将以可读形式保存在本地文件中
- 这是一个本地便利功能，不是高强度安全存储方案
- 用户取消勾选后，文件应被删除或清空为无 remembered login 状态

### 5.3 建议新增服务

新增固定服务模块：

- `services/login_preferences.py`

职责：

- 读取 remembered login
- 保存 remembered login
- 清除 remembered login
- 在删除账号时按用户名清理 remembered login

这样可以避免把 JSON 文件读写逻辑散落在 `LoginWindow`、用户中心等多个 UI 类中。

### 6.3 职责划分

`ui/theme.py`：

- 提供统一 QSS 主题
- 暴露固定入口 `apply_theme(app)`
- 尽量通过全局样式 + `objectName` 定点增强完成主题控制

`services/login_preferences.py`：

- 负责 remembered login 的文件读写
- 提供安全兜底，文件损坏时返回空状态而不是直接让 UI 崩溃

`ui/login_window.py`：

- 新增记住密码复选框
- 初始化时加载 remembered login
- 登录成功后按勾选状态保存或清理 remembered login
- 页面结构升级为单卡片布局

`ui/register_dialog.py`：

- 跟随新主题统一样式

`ui/password_field.py`：

- 替换当前容易受编码影响的按钮文案
- 统一输入框与切换按钮样式

`main.py`：

- 在 `QApplication` 创建后、窗口显示前统一应用主题

## 7. 数据流

### 7.1 应用启动到登录页展示

1. `main.py` 创建 `QApplication`
2. 应用全局主题
3. 构造 `LoginWindow`
4. `LoginWindow` 初始化时读取 remembered login
5. 若有 remembered login，则回填用户名和密码
6. 默认将“记住密码”设为勾选状态

### 7.2 登录成功

1. 用户点击“登录”
2. `LoginWindow` 调用现有 `authenticate_user`
3. 若认证成功：
   - 若勾选“记住密码”，则保存用户名、密码和更新时间
   - 若未勾选，则清除 remembered login
4. 发送 `login_succeeded`
5. 主窗口显示

### 7.3 删除账号

1. 用户中心执行删除账号
2. 现有账号删除流程成功后
3. 若 remembered login 中的用户名与当前删除账号一致，则同步清理 remembered login
4. 退出登录并回到登录页

## 8. 错误处理

- remembered login 文件不存在：按空状态处理
- remembered login 文件损坏或字段不合法：忽略该文件并按空状态处理
- remembered login 清理失败：不阻断 UI 主流程，但应给出统一错误提示或静默回退到空状态
- 登录失败：继续沿用当前错误提示策略，不暴露额外敏感信息
- 主题样式加载失败：应用仍应可启动，最差回退到默认 Qt 样式

## 9. 测试与验证范围

至少验证以下场景：

- 首次打开登录页时，“记住密码”默认勾选
- 勾选状态下登录成功，重新启动应用后能回填用户名和密码
- 取消勾选后，本地 remembered login 被清理
- 登录失败不会错误写入 remembered login
- 删除账号后，对应 remembered login 会被同步清理
- 登录页、注册弹窗在新主题下正常显示
- 主界面标签页切换、数据导入、预测、历史记录、用户中心功能不受主题改造影响

建议新增测试类型：

- `services/login_preferences.py` 的单元测试
- remembered login 删除逻辑的服务级测试
- 主题应用后的基本启动/导入测试（可先做轻量 smoke test）

## 10. 实施建议

按以下 3 步落地：

1. 先加 remembered login 服务与登录页交互
2. 再引入全局主题层并改造登录页/注册弹窗
3. 最后统一主界面与各页面控件样式，补齐验证

这样可以先把功能闭环做好，再处理视觉统一，降低回归风险。

## 11. 验收标准

当以下条件同时满足时，本次需求视为完成：

- 登录页呈现为方案 B 的单卡片数据科技风
- 注册弹窗与主界面风格统一
- 用户重新打开应用时，用户名和密码会自动回填
- 应用不会自动登录，仍需手动点击“登录”
- 取消“记住密码”后，本地 remembered login 被清除
- 删除账号后不会残留对应的 remembered login 数据
- 原有注册、登录、改密、预测、记录查看等功能保持可用
